Git Bash commands:


git --version	: shows the version of git
git clone url   : clones the repository on local machine
git status      : shows the status of the repository code
git add filename    : adds the new or changed file to your working directory to the Git staging area
git commit -m "message"     : commits the changes with a message
git add .	: adds all the files in the current directory to the Git staging area

git init    : used to create a new local repository
git remote add origin url   : adds a remote repository
git remote -v   : shows the remote repository url(to verify remote)


git branch     : to check the branch (github default branch is master)
git branch -M main     : to rename the branch
git branch -a	: shows all the branches in the repository

git checkout branch-name   : to navigate to a different branch
git checkout -b branch-name   : to create a new branch and navigate to it
git branch -d branch-name   : to delete a branch

-----------------------------

git push origin main    : pushes the local repo changes to the remote repository
git push -u origin main    : pushes the local repo changes to the remote repository and sets the upstream tracking information  (creates  short form for future)
git pull origin main    : pulls the remote repo changes to the local repository

--------------------------------------------------------
UNDO CHANGES:
(case1)= staged changes
        git reset filename : unstage the file
        git reset : unstage all files
(case2)= commited changes(for 1 commit)
        git reset HEAD-1 : undo the last commit
(case3)= commited changes(for multiple commits)
        git reset commit-hash : undo the commit with the given hash
        git reset --hard commit-hash : undo the commit with the given hash and discard all changes
